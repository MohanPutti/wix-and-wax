## 1 — Schema migration (run once per schema change)

The Prisma schema source of truth is **`core/prisma/schema.prisma`**.
`server/prisma/schema.prisma` is generated by the Dockerfile — never edit it directly.

### 1a. Push schema to production RDS from your local machine

```bash
# Make sure server/.env has DATABASE_URL pointing to RDS
# Check it:
grep DATABASE_URL server/.env

# If it points to localhost, temporarily set the RDS URL:

# Run prisma db push (non-interactive, safe for CI)
cd server
cp ../core/prisma/schema.prisma prisma/schema.prisma
npx prisma db push --schema=prisma/schema.prisma
```

- `colors`        — master color list with optional hex values

## 2 — Data migration (run once, converts existing product variants)

All 129 existing products were already migrated in the **local dev database**.
If the production database still has the old variant format (`options: { size: "Small (100g)" }`),
run the migration script against production RDS.

### 2a. Check if migration is needed

```bash
# On EC2:
ssh -i ~/.ssh/portfolio-parser-key.pem ubuntu@13.233.190.247

# Inside EC2, check a sample variant:
docker exec $(docker ps -q) node -e "
const { PrismaClient } = require('@prisma/client');
const p = new PrismaClient();
p.productVariant.findFirst().then(v => { console.log(v?.options); p.\$disconnect(); });
"
```

If output shows `{ "size": "Small (100g)" }` → migration needed.
If output shows `{ "base": "Jar", "size": "100g" }` → already done, skip step 2.

### 2b. Run the migration

Create a temporary file `migrate-prod.mjs` locally:

```js
// migrate-prod.mjs — run with: node migrate-prod.mjs
import { PrismaClient } from '@prisma/client';

const SIZE_MAP = {
  'Small (100g)':  '100g',
  'Medium (200g)': '200g',
  'Large (350g)':  '350g',
};

const prisma = new PrismaClient();

async function main() {
  const variants = await prisma.productVariant.findMany();
  let migrated = 0, skipped = 0;

  for (const v of variants) {
    const opts = v.options;
    if (!opts || opts.base) { skipped++; continue; }       // already migrated

    const oldSize = opts.size || '';
    const newSize = SIZE_MAP[oldSize] || oldSize.replace(/.*\((.+)\).*/, '$1').trim();
    const newBase = 'Jar';
    const newName = `${newBase} - ${newSize}`;

    await prisma.productVariant.update({
      where: { id: v.id },
      data: {
        options: { base: newBase, size: newSize },
        name: newName,
      },
    });
    migrated++;
  }

  console.log(`Done. Migrated: ${migrated}, Skipped (already done): ${skipped}`);
}

main().finally(() => prisma.$disconnect());
```

Then run it pointing at production RDS:

```bash
DATABASE_URL="mysql://waxadmin:<PASSWORD>@wix-and-wax-db.cpaao8eykg3n.ap-south-1.rds.amazonaws.com:3306/wix_and_wax" \
  node migrate-prod.mjs
```

## 3 — Build & deploy backend (Docker)

```bash
# 1. Build Docker image locally
docker build -t wix-and-wax-backend .

# 2. Transfer image to EC2 and reload container (single command)
docker save wix-and-wax-backend | gzip | \
  ssh -i ~/.ssh/portfolio-parser-key.pem ubuntu@13.233.190.247 \
  "gunzip | docker load && cd /app/wix-and-wax && sudo docker compose up -d --force-recreate"

# 3. Verify backend is healthy
ssh -i ~/.ssh/portfolio-parser-key.pem ubuntu@13.233.190.247 \
  "curl -s http://localhost:3001/api/health"
```

## 4 — Build & deploy frontend (static files)

```bash
# 1. Build frontend locally
cd client
npm run build     # runs: vite build (TS errors skipped by vite, output to dist/)

# 2. Upload to EC2
rsync -avz --delete -e "ssh -i ~/.ssh/portfolio-parser-key.pem" \
  dist/ \
  ubuntu@13.233.190.247:/var/www/wix-and-wax/client/dist/

cd ..
```

> `rsync --delete` removes stale files from the server so old JS chunks don't linger.
